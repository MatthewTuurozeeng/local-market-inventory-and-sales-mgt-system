# Figma Design Guide — Local Market Inventory & Sales Management System

This guide explains what your Figma file should look like and how to make it professional.

## 1) Figma File Structure (Pages)
1. **00 Cover / Index**
2. **01 Design System**
3. **02 User Flows**
4. **03 Desktop Screens**
5. **04 Mobile Screens**
6. **05 Prototype**

## 2) Design System (Professional Requirements)
### Colors
- **Primary:** #0F766E (teal)
- **Secondary:** #1D4ED8 (blue)
- **Success:** #16A34A
- **Warning:** #F59E0B
- **Error:** #DC2626
- **Neutrals:** #111827, #4B5563, #9CA3AF, #E5E7EB, #F9FAFB

### Typography (Suggested)
- **Font:** Inter
- **H1:** 32/40, SemiBold
- **H2:** 24/32, SemiBold
- **H3:** 20/28, SemiBold
- **Body:** 16/24, Regular
- **Caption:** 12/16, Regular

### Spacing Scale
4, 8, 12, 16, 24, 32, 40

### Components to Build
- Buttons: Primary, Secondary, Ghost, Disabled
- Inputs: Text, Number, Password, Select, Search
- Cards: KPI Card, List Card, Product Card
- Chips: Status (Pending/Ready/Completed)
- Alerts: Success/Warning/Error
- Table: Inventory and Orders list
- Navigation: Sidebar (desktop), Bottom nav (mobile)

## 3) User Flows (Minimum)
- **Login → Dashboard**
- **Inventory → Add Product → Inventory Updated**
- **Sales Entry → Daily Summary**
- **Orders → Status Update → Customer Tracking**

## 4) Core Screens (Desktop)
1. **Login / Register**
2. **Dashboard (KPIs + Quick Actions)**
3. **Inventory List**
4. **Add/Edit Product**
5. **Sales Entry**
6. **Sales Summary**
7. **Orders List**
8. **Order Details**
9. **Notifications**
10. **Analytics Dashboard**

## 5) Core Screens (Mobile)
- Inventory List (mobile)
- Add Product (mobile)
- Sales Entry (mobile)
- Order Tracking (mobile)

## 6) Layout & Grid
- **Desktop:** 12‑column grid, 1200px content width
- **Mobile:** 360px frame, 8‑column grid
- Use **Auto Layout** for all cards and lists

## 7) Visual Style Guidance
- Minimal, clean UI with generous white space
- Use color sparingly for emphasis
- Consistent component spacing and alignment

## 8) Prototyping
Create a clickable flow that covers:
- Login → Dashboard
- Inventory → Add Product → Back to Inventory
- Sales Entry → Sales Summary
- Orders → Order Details → Status Updated

## 9) Naming Convention (Professional Standard)
Format: `Section / Screen / Variant`
- Example: `Inventory / List / Desktop`
- Example: `Orders / Detail / Mobile`

## 10) Definition of “Professional” (Checklist)
- Reusable components with Auto Layout
- Design system documented
- Grid‑aligned screens
- Consistent typography and spacing
- Prototype works end‑to‑end
